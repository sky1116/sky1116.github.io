我的 MVVM 之路 (一) - Data Binding

Posted on 2019-03-16 | Post modified: 2020-12-20 | In android

這一系列的文章將以 MVVM 為目標，逐步導入 Android Architecture Components 的方式來講解重構演進的過程。

認識 MVVM

MVVM (Model–View–ViewModel) 是由 Martin Fowler 的 PM (Presentation Model) 衍生進而達成 SOC (Separation Of Concerns) 原則的一種設計模式。

View

單純的 UI 元件，使用觀察者模式與 ViewModel 綁定，當 properties 更新時，View 會收到更新通知做出對應的行為，內部只有處理畫面的邏輯。

ViewModel

主要暴露 properties 跟 commands 的接口，做為 View 跟 Model 中間的橋梁，儲存 View 顯示的資料來源，可獨立存在，不直接對 View 做操作，也不應該知道 View 的存在。

Model

資料存取層，包含本地端及遠端資料的存取，商業邏輯也寫在這裡，除了自己以外不應該知道 View 跟 ViewModel 的存在。

Model

ViewModel

ViewModel 有一個取得資料的 command 接口，內部呼叫 Model 來取資料，目前尚未與 View 做綁定。
MainViewModel.java

View

Activity 只保留單純的 View。
MainActivity.java

類別建立後，接下來用 Data Binding 來將 View 跟 ViewModel 綁定。

Data Binding

在 build.gradle 中啟用 dataBinding。
app\build.gradle

在原本的 activity_main 的最外層加上 (在 Root View 按 Alt + Enter -> Convert to data binding layout)，並在 裡面宣告要跟 View 綁定的 ViewModel
activity_main.xml

接下來 Rebuild Project，就會自動產生 binding 類別，假設 layout 的名稱是 activity_main.xml，在編譯過後就會自動產生 ActivityMainBinding 這個類別，然後用這個類別來跟 View 做綁定。
MainActivity.java

從 View 往下延伸，接著是 ViewModel，前面說過綁定是透過觀察者模式，所以在這裡將變數宣告為 Observable(註)。
MainViewModel.java

註：ObservableBoolean、ObservableByte、ObservableShort、ObservableInt、ObservableLong、ObservableChar、ObservableFloat、ObservableDouble、ObservableParcelable、ObservableArrayMap、ObservableArrayList

最後在 xml 裡面使用 @{ } 的方式將 ViewModel 的值設定給 View。
