我的 MVVM 之路 (六) - RecyclerView Adapter

Posted on 2019-04-05 | Post modified: 2020-12-20 | In android

前一篇開始導入了 RecyclerView，這一篇來試著改良一下 Adapter 吧。

CommonAdapter

觀察一下目前的 Adapter，可以發現，Adapter 的變動因素只有兩個 - UserListItemBinding 和 UserInfo，採用 Liskov 原則讓他變成通用的 RecyclerView Adapter。

先將原本 UserAdapter.java 複製改名為 CommonAdapter.java，接下來都是對這個檔案做修改。

Data Binding 產生的實例都是繼承自 ViewDataBinding，所以把 Adapter 裡面的 UserListItemBinding 都替換成 ViewDataBinding。

ViewHolder

首先先來處理 ViewHolder。

這時候，在 bind() 裡面 setUserInfo() 會產生錯誤，前面說過這個方法是根據 xml 裡面 variable 的 name 所產生的，但是 ViewDataBinding 並沒有跟任何 layout 綁定，自然不會產生任何 set 的方法，這時候可以使用 setVariable(int variableId, @Nullable Object value) 來替代，在那之前，先來幫 variable 的 name 改個通用一點的名字，因為目標是讓 Adapter 成為通用的，那 name 就使用統一的名稱，這裡把他改為 item (可自行改名，專案統一就好)。

接下來回到 setVariable，裡面的參數 variabled 對應的是 variable 內的 name，可以使用 BR 來取得專案內 variable 的 name。

這裡改完可以先重新 build 專案，檢查是否有錯誤。

onCreateViewHolder

接著是 onCreateViewHolder 這個方法，這裡稍微解釋一下，因為我們的 UserListItemBinding 是根據 user_list_item.xml build 後產生的，所以已經跟 layout 綁定了，但是 ViewDataBinding 只是一個單純的抽象類別且並沒有 inflate 方法，這裡就使用 DataBindingUtil.inflate 的方法來產生。

UserInfo

UserInfo 只是一個單純的資料類別，需要提煉一個父類別，配合 DiffUtil 的差異比對，建立一個 Comparable 的 interface 讓 UserInfo 來實作。

接著把 DiffUtill 裡面的比對搬到 UserInfo 的實作。

UserInfoDiffCallBack

修改原本的 UserInfoDiffCallBack，除了替換 UserInfo，也把變數改成通用的命名。

最後把 Adapter 剩下還有 UserInfo 的地方都一併修改，然後重新 build 測試。

layout resource

因為 onCreateViewHolder 的地方有做了調整，導致必須要認識 item 的 layout，這裡大致上有兩種做法，一種是在建構式新增參數，一種是使用抽象方法取得 layout 讓子類別去實作，在這會使用第一種作法，畢竟以通用 Adapter 為目標，希望不需要再另外繼承。

接著到 MainActivity.java 把 UserAdapter 改成 CommonAdapter 就完成了。
